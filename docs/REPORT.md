## 1. Problem Definition

We receive a **mixed signal** composed of four superimposed sine waves at 1, 5, 10, and 20 Hz,
each with random amplitude jitter (σ=0.05) and phase jitter (σ=0.1), plus additive Gaussian
noise (σ=0.05). The task is to **extract the 10 Hz sine wave** from each 10-sample window of
the mixture using three families of neural networks.

This is a supervised regression problem: given a noisy window `x ∈ ℝ¹⁰`, predict the clean
target window `y ∈ ℝ¹⁰` corresponding to the 10 Hz component.

---

## 2. Data Generation

Each sample is generated by:

1. Summing sine waves: `s(t) = Σ Aᵢ · sin(2π fᵢ t + φᵢ)` for fᵢ ∈ {1,5,10,20}
2. Adding noise: `x(t) = s(t) + ε`, ε ~ N(0, 0.05²)
3. Sliding a window of 10 samples at 200 Hz sampling rate
4. Storing the corresponding 10 Hz component alone as the target

10,000 windows were generated, split 70/15/15 into train/val/test.

Per the Nyquist theorem, a 200 Hz sampling rate supports frequencies up to 100 Hz — well above
our highest frequency of 20 Hz.

---

## 3. Model Design

A **frequency label** (one-hot vector of length 4) is passed alongside the input to each model,
telling the network which frequency to extract. This design allows a single trained model to
extract any of the four frequencies.

### MLP
Fully connected: `[concat(x, label)] → 1024 → 512 → 256 → W` with LeakyReLU activations.
Treats the window as a flat vector; no explicit temporal structure.

### RNN
Bidirectional vanilla RNN processes the sequence step-by-step. The final hidden state is
concatenated with the frequency label and projected to the output window. Limited by gradient
vanishing in longer sequences.

### LSTM
Same structure as RNN but uses LSTM cells with input/forget/output gates, which alleviate
gradient vanishing and allow better modelling of temporal dependencies.

## 5. Results

| Model | MSE    | MAE    | R²     |
|-------|--------|--------|--------|
| MLP   | 0.0582 | 0.1811 | 0.8836 |
| LSTM  | 0.0568 | 0.1742 | **0.8863** |
| RNN   | 0.1828 | 0.3373 | 0.6343 |

**LSTM** performs best overall (R²=0.886), narrowly outperforming MLP (R²=0.884).
Both substantially outperform the vanilla **RNN** (R²=0.634), confirming that gating
mechanisms are important for this type of signal extraction task.

---

## 6. Analysis

**Why does LSTM beat MLP?**  
LSTM can exploit the sequential structure of the sine wave — adjacent samples are
correlated in a way that a flat MLP ignores. The gating mechanism allows it to selectively
retain phase and amplitude information across the window.

**Why does RNN lag behind?**  
The vanilla RNN suffers from gradient vanishing even over the short 10-sample window because
it lacks gating. Its hidden state is overwritten at each step without selective memory. The
bidirectional architecture helps, but is not sufficient to close the gap with LSTM.

**Why is MLP competitive with LSTM?**  
With only 10 time-steps, temporal structure is weak. A well-regularised MLP with sufficient
capacity can learn the mapping `(noisy window, frequency label) → clean window` from
statistical co-occurrence alone, without needing sequential processing.

---

## 7. Conclusion

All three models successfully learn to suppress noise and extract the target sine wave, but
performance varies significantly. LSTM is the recommended architecture for this task. Future
improvements would include attention mechanisms on the RNN output, a larger window size for
richer temporal context, and ensembling LSTM with MLP for an estimated R² ≈ 0.91.
